# bracketball App Store Screenshot Prep

Status: working checklist for the screenshot phase only

Do not use this as the main priority list. First use [App Store Review Readiness](./app-store-review-readiness.md) to decide whether you are actually ready to start screenshots.

Use this before you start taking App Store screenshots on your iPhone.

The goal is to make the listing feel cohesive, polished, and unmistakably mobile.

## One account strategy

Use one dedicated demo/review account for all screenshot capture.

Why:
- the visuals stay consistent
- the same pools and drafts appear across shots
- you avoid awkward jumps between empty and full states

Ideal demo account setup:
- at least 1 joined pool with a strong name
- at least 1 saved draft
- at least 4-6 leaderboard entries in the featured pool
- a bracket with enough advancement visible to read well
- profile fully filled out with avatar, full name, favorite team, and push enabled

## Best featured pool

Pick one primary pool to anchor most of the screenshots.

That pool should have:
- a clean, readable name
- real-looking standings
- no broken or placeholder data
- enough members to make the leaderboard feel competitive
- a bracket that looks meaningfully populated

Good pool qualities:
- short name
- readable score differences
- realistic player names
- no obvious testing artifacts

Avoid using a featured pool if:
- the leaderboard is too empty
- there are only 1-2 entries
- names look fake or repetitive
- the bracket is mostly blank

## Screen-by-screen prep

### Home

Target look:
- app-like shell visible
- current tournament context visible
- not too much dead space

Before capture:
- make sure no error or loading states are present
- make sure the mobile nav looks stable
- avoid partial modals or temporary banners

### Pools

Target look:
- one joined pool clearly visible
- `discover & join` visible
- clean card spacing

Before capture:
- make sure pool names look polished
- avoid clutter from too many mediocre pools
- use the account with the strongest joined-pool lineup

### Leaderboard

Target look:
- pool name visible
- top standings readable
- `Share Invite` visible if it fits cleanly

Before capture:
- use your strongest pool
- make sure at least 4-6 rows look good
- avoid ties or score weirdness if it makes the board feel flat

### Bracket

Target look:
- progression clearly visible
- enough of the bracket visible to understand advancement

Before capture:
- use a state where the bracket is populated
- avoid extremely zoomed-out unreadable captures
- avoid a nearly empty early-stage bracket if a later-state one looks stronger

### Profile

Target look:
- avatar visible
- profile data filled out
- push notifications section visible

Before capture:
- use a clean avatar
- make sure name/team are filled
- enable push notifications on that device if the enabled state looks stronger

### Draft flow

Target look:
- this app clearly includes actual drafting, not just viewing standings

Before capture:
- pick the draft screen with the cleanest visual density
- avoid awkward edge cases or empty draft states
- prefer a view where picks are obvious at a glance

## Cleanup before capture

Remove or avoid:
- temporary debug UI
- admin-only controls if they clutter the shot
- test/error banners
- empty states unless they are intentionally beautiful
- half-finished seeded data
- inconsistent names or obviously fake entries

## Capture order

Take screenshots in this order:
1. Home
2. Pools
3. Leaderboard
4. Bracket
5. Profile
6. Draft flow

Why this order:
- it tells the cleanest user story
- it front-loads the strongest app-wide screens
- it keeps the competitive value proposition obvious

## Final capture checklist

- [ ] Captured from the real iPhone app
- [ ] Same account used across all shots
- [ ] Same visual theme across all shots
- [ ] Featured pool looks polished
- [ ] No debug or test UI appears
- [ ] Profile is fully filled out
- [ ] Push section looks intentional
- [ ] Draft screen is understandable
- [ ] Bracket is populated enough to read
- [ ] Leaderboard has meaningful standings

## What to fix before shooting

If any of these are true, fix them first:
- the Pools screen still feels cluttered
- the featured pool leaderboard is too empty
- the bracket looks sparse or confusing
- the profile screen looks unfinished
- the draft screen is too dense to parse quickly

If one planned screenshot is weak, do not force it. Replace it with the strongest real screen in the app that still helps tell the story.
